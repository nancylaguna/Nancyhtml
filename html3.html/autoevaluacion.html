Ejercicios de autoevaluación

Complete las siguientes oraciones:

  1. El tipo input color permite al usuario introducir un color. Al momento de escribir este libro, la mayoría de los navegadores despliegan el tipo input color como un campo de texto en donde el usuario puedeintroducir un ----- código hexadecimal -----.
  2. El atributo ----- placeholder ----- nos permite colocar texto temporal en un campo de texto.
  3. Si desea anular la validación, puede agregar el atributo formnovalidate al tipo input ----- submit ----- .
  4. El atributo ----- required ----- obliga al usuario a introducir un valor antes de enviar el formulario.
  5. Por lo general se muestra el control ----- spinner ----- para el tipo input number e incluye sólo los números válidos.
  6. El tipo input ----- time ----- permite al usuario introducir una hora, minuto, segundo y fracción de segundo.
  7. El elemento ----- datalist ----- proporciona opciones de entrada para un elemnto input text
  8. el elemento ----- article ----- describe contenido que está separado del contenido principal de la página y que posiblemente podría utilizarse o distribuirse en otra parte, como un artículo de noticias, mensaje de foro o entrada de blog.
  9. El elemento ----- footer ----- describe el texto que por lo general aparece en la parte inferior del contenido o de un elemento section.
  10. El elemento ----- wbr ----- indica el lugar apropiado para dividir una palabra cuando el texto se ajusto en varios líneas.

Indique si cada uno de los siguientes enunciados es verdadero o falso. Si es falso, explique por qué.

  1. Cualquier tipo input particular de formulario HTML5 debe desplegarse de manera idéntica en todos los navegadores compatibles con HTML5. ----- Falso. El despliegue de los tipos input puede variar de un navegador a otro. -----
  2. Cuando se coloca el enfoque en el campo de texto (es decir, si el cursor está en el campo de texto), se envía el texto del atributo placeholder al servidor.----- Falso. Cuando se coloca el enfoque en el campo de texto, e texto de placeholder desaparece. No se “envía” cuando el usuario hace clic en el botón Enviar (a menos que el usuario escriba el mismo texto). -----
  3. No es necesario incluir el atributo autofocus en sus formularios.----- Verdadero -----
  4. Los nuevos tipos input de HTML5 tienen validación automática del lado del cliente, con lo cual se elimina la necesidad de agregar secuencias de comandos (script) complicadas a sus formularios para validar la entrada del usuario y se reduce la cantidad de datos inválidos que se envían.----- Verdadero -----
  5. El tipo input range se valida automáticamente de manera intrínseca cuando el navegador lo despliega como un control deslizante, ya que el usuario no puede mover el control fuera de los límiites del valor mínimo o máximo.----- Verdadero -----
  6. HTML5 cuenta con validación automática para el tipo input tel.----- Falso. La longitud y el formato de los números telefónicos varía de manera considerable con base en la ubicación, por lo que la validación se vuelve bastante compleja, así que HTML5 no cuenta con validación automática para el tipo input tel. Para asegurar que el usuario introduzca el número en un formato apropiado, usamos el atributo pattern.-----
  7. Si el usuario ntroduce un URL con un formato inapropiado en un tipo input url, no se validará. HTML5 no valida si el URL introducido existe en realidad.----- Verdadero -----
  8. El elemento nav muestra un menú desplegable de hipervínculos.----- Falso. El elemento nav agrupa vínculos de navegación. -----
  9. El elemento header puede usarse sólo una vez en una página.----- Falso. El elemento header puede usarse varias veces en una página y a menudo incluye encabezados de HTML (< hl > a < h6 >). -----
  10. Los elementos nav pueden anidarse en un elemento aside.----- Verdadero -----
  11. Podemos usar el elemento brk para evitar dividir palabras en lugares incorrectos.----- Falso. Puede usar el elemento wbr para evitar dividir las palabras en lugares incorrectos. -----
